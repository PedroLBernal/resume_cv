def multi_armed_bandit(num_games=1000, epsilon=0.1, verbose=False, bandits = [0.1, 0.05, 0.03, 0.7, 0.02]):
    total_reward = 0
    acum_reward_bandit = np.zeros(len(bandits))  # numerador
    num_selected_bandit = np.zeros(len(bandits)) # denominador
    q_bandits = np.zeros(len(bandits))            # Q(a)
    
    if verbose:
        print("Initial Bandits Distribution\n  {}".format(bandits))
    
    for game in range(0,num_games):
        
        old_q_bandits = q_bandits.copy()
        
        if np.random.random() < epsilon:
            bandit = np.random.randint(len(bandits)) # Exploro
        else:
            bandit = np.random.choice(np.flatnonzero(q_bandits == q_bandits.max())) # Exploto 
            
        # Obtengo el reward
        reward = 1 if (np.random.random() < bandits[bandit]) else 0
        
        # Actualizo reward total
        total_reward += reward
        
        # Actualizo valor (Q) de la imagen
        acum_reward_bandit[bandit] += reward
        num_selected_bandit[bandit] += 1
        q_bandits[bandit] = acum_reward_bandit[bandit] / num_selected_bandit[bandit]
        
        if verbose:
            print("\nGAME {game}\n  Old Q_Bandits = {old_q_bandits}\n  Selected Bandit = {bandit} \
                  \n  Reward = {reward}\n  Q_Bandits = {q_bandits}"
                  .format(game=game+1, old_q_bandits=old_q_bandits, bandit=bandit, 
                          reward=reward, q_bandits=q_bandits))
    
    return bandits, total_reward, q_bandits, num_selected_bandit


# Generador de lista de nmeros aleatorios; se "trampea" para que no sea superior a 0.75
def generar_lista(_n=5):
    numbers = np.random.exponential(scale=0.1, size=_n)
    numbers = np.round(numbers, 2)
    
    return list(numbers)

El mtodo epsilon-greedy es una estrategia muy comn en el algoritmo del bandido multibrazo, donde psilon define la probabilidad con la que el algoritmo explora (elige una imagen al azar, independientemente de cul sea en ese momento la que ms clicks consigue), siendo 1 - psilon la probabilidad con la que el algoritmo explota (es decir, elige la imagen ms valorada en ese momento para mostrar).

Si se prueban diferentes valores de psilon para el mismo conjunto de imgenes, vemos que en el rango de psilon = [0.05, 0.2] se obtienen los mayores nmeros de clicks para la ltima foto (que es, a la postre, la que tiene mayor tasa de xito en nuestra lista de ratios generada aleatoriamente), todos por encima de los 600 clicks, siendo el mximo nmero de clicks el alcanzado por la prueba realizada para psilon igual a 0.1. Cuanto menos se 'imprime' la imagen 5, menor es el nmero total de clicks que se consiguen.

Esto se puede ver en la ltima prueba, cuando se pas a la funcin un valor para psilon muy elevado (0.5, es decir, para cada impresin, el algoritmo tiene la misma probabilidad de explorar o actuar de manera aleatoria, que de explotar o mostrar la 'mejor' imagen hasta ese momento). En esta prueba, el nmero de veces que se imprimi la ltima imagen fue de 612 veces, bastante menos que en la segunda prueba con menor nmero de impresiones (797) y se obtuvieron casi 100 clicks menos que en la segunda prueba con menor nmero de clicks (en torno a un 15-16% menos clicks).

En conclusin, psilon debe estar en un rango de valores entre 0.05 y 0.2 (siendo 0.1 el que maximiz la recompensa en las pruebas).

3. Poniendo una ratio de explotación del 100%, ¿Qué resultado se obtiene?¶

Un ratio de explotacin del 100% implica un psilon = 0, ya que no se explorara nada y todas las veces se explotara. En la prctica, sto significa que el algoritmo siempre elegir la opcin con mayor valor de Q, en este caso, la imagen con mayor ratio de clicks/impresiones, y nunca mostrar una imagen al azar (exploracin). En el momento inicial, como ninguna imagen ha sido mostrada, todas tienen un Q igual a 0, por tanto, todas son "igual de buenas", no hay ninguna mejor ni peor, por lo que el algoritmo seleccionar una al azar. En el momento en que una de las imgenes obtenga su primer click, su Q pasar a ser mayor que 0, por lo que pasar a ser la mejor imagen y, desde entonces y hasta haber mostrado las 1000 impresiones, el algoritmo slo seleccionar a sta, puesto que nunca probar otras opciones y, por tanto, no dar al resto de imgenes la oportunidad de mejorar el valor de su Q.
